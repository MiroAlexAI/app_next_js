# AI Reader - Парсер статей

## ✅ Реализовано

### Функциональность
Приложение парсит англоязычные статьи и извлекает:
- **Заголовок** (title)
- **Дату публикации** (date)
- **Основной контент** (content)

### Технический стек
- **Next.js 16** (App Router)
- **Tailwind CSS 3.4** (стилизация)
- **Cheerio** (парсинг HTML)
- **Axios** (HTTP-запросы)

### API Endpoint
`POST /api/parse`

**Request:**
```json
{
  "url": "https://example.com/article"
}
```

**Response:**
```json
{
  "date": "2025-12-20",
  "title": "Article Title",
  "content": "Full article content..."
}
```

### Интерфейс

1. **Поле ввода URL** - вставьте ссылку на статью
2. **3 кнопки действий:**
   - **"О чем статья?"** - показывает заголовок, дату и фрагмент контента
   - **"Тезисы"** - выводит полный JSON с данными
   - **"Пост для Telegram"** - форматирует данные для соцсетей

3. **Блок результата** - отображает распарсенные данные с возможностью копирования

### Парсинг контента

Приложение использует множественные селекторы для поиска контента:
- `<article>`
- `[role="main"]`
- `<main>`
- `.post-content`, `.article-content`, `.entry-content`
- `.content`, `.post`, `.article-body`
- `#content`, `.story-body`, `.post-body`

Автоматически удаляются:
- Скрипты и стили
- Навигация, хедер, футер
- Реклама и социальные кнопки

### Запуск

```bash
# Development
npm run dev

# Production build
npm run build
npm start
```

**URL:** http://localhost:3000

### Пример использования

1. Откройте http://localhost:3000
2. Вставьте URL статьи (например, с Medium, Dev.to, или любого блога)
3. Нажмите любую из трех кнопок
4. Результат появится в блоке ниже

### Ограничения

- Контент обрезается до 5000 символов для оптимизации
- Timeout запроса: 10 секунд
- Требуется доступность целевого сайта (без CORS-блокировок на стороне сервера)

### Следующие шаги (опционально)

Для интеграции с AI (GPT, Claude и т.п.):
1. Создать API endpoint `/api/ai-process`
2. Передавать распарсенный контент в LLM
3. Получать обработанный результат (саммари, тезисы, пост)
